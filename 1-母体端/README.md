# 1-母体端

母体核心区，放 MBclaw 的运行时、记忆、工具调度、Gateway、思考层、规划层与执行编排。

## 这个目录负责什么
- 让消息进入后能被统一理解。
- 让记忆、工具、Agent、Runtime 形成闭环。
- 让母体只做系统核心，不做平台杂务。

## 子目录
- 1-记忆系统：短期记忆、长期记忆、项目记忆、决策记忆、经验沉淀。
- 2-工具层：统一工具注册、调用、结果回传、工具索引。
- 3-Agent：执行型、规划型、评论型、协作型 Agent。
- 4-Gateway：平台入口、消息归一、标准化输入输出。
- 5-思考与PRT：规划、推理、反思、施工记录、任务提示。
- 6-Runtime：事件循环、状态机、任务调度与执行编排。

## 设计原则
- Gateway 先行，母体后进。
- 记忆、工具、Agent、Runtime 分层，不互相吞职责。
- 主线实现要能接手、能回溯、能继续施工。
