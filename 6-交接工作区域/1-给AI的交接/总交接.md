## 已完成的 MBclaw 地基改造

### 1. 统一模型入口
- `GET /health`（原有）
- `GET /v1/health`（新增，匹配 Mother 以 `/v1` base URL 调用 `Sub2APIClient.health()`）
- `POST /v1/chat/completions`、`POST /v1/embeddings`（sub2api 原生）

### 2. MBclaw 管理命名空间 `/api/v1/admin/mbclaw/*`
- `POST /api/v1/admin/mbclaw/upstream-tokens`：接收 MBclaw 用户上传 key，写入 sub2api account，返回脱敏 account
- `POST /api/v1/admin/mbclaw/token-pool-modules`：注册 miclaw 免费代理、商业中转站等外部 token 池模块
- `GET /api/v1/admin/mbclaw/status`：向 Mother 汇报地基入口、控制入口、token 来源、倍率/并发承接状态

### 3. 细节功能（倍率/并发/限额）的现有承接点
- 单用户并发：`users.concurrency`
- 单用户倍率：group `rate-multipliers` + `rpm_override`
- 单 key 限额：API key `quota`、`rate_limit_5h/1d/7d`
- 单模型倍率：channel `model_pricing` + `model_mapping`
- 单上游 key 并发/倍率：account `concurrency` / `rate_multiplier`（默认 1）
- 调度/failover/cooldown：sub2api 原生 scheduler + failover_loop

### 4. 管理面板
- sub2api 原生管理面板（accounts/groups/channels/API keys/usage/ops）
- 新增侧边栏 "MBclaw 地基" 页面（`/admin/mbclaw`），可查看状态、上传 key、注册 token 池模块
- MBclaw 内置 token 管理面板（预留），商业化中转站和 miclaw 免费代理承接路径已打通

### 5. 母体掌控入口
- Mother 通过 `Sub2APIClient` 以 `/v1` base URL 调用 sub2api
- `GET /api/v1/admin/mbclaw/status` 向 Mother 实时汇报全量地基状态

## 风险点
- 编译需要 `-tags embed` 才能内嵌前端；不加此 tag 前端页面返回 404
- 二进制需要从 `config.yaml` 和 `.installed` 所在目录启动，或用 `DATA_DIR` 环境变量
- Go 1.26.5 版本是硬性要求
- pnpm 11 需先 `pnpm approve-builds esbuild vue-demi`
- 测试机无 systemd，需手动 `nohup &` 启动
- pkill -f sub2api-server 会杀掉 SSH 会话，建议用 `kill <PID>`
