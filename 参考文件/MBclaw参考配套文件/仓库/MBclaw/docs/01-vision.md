# 01 — 项目愿景与目标

## 为什么做 MBclaw

市面上的 AI 大多数只能记住当前上下文。对话结束，一切遗忘。

MBclaw 的目标是：

> 让 AI 能够记住整个项目的发展过程，在数月后继续项目时仍然能够快速恢复状态。

## 不仅要记住聊天记录

MBclaw 的记忆维度远超普通聊天记录：

- **项目目标** — 这个项目要达成什么
- **成功方案** — 哪些方案经过验证可行，为什么
- **失败方案** — 哪些方案走不通，原因是什么
- **经验总结** — 从过往对话中提炼的经验
- **关键词** — 自动提取的技术关键词索引
- **后续计划** — 下一步要做什么

## 设计原则

1. **单用户先行** — MVP 阶段支持单用户，但数据模型预留多用户扩展
2. **会话即边界** — 每次对话 = 一个 Session，总结/关键词/经验均以 Session 为触发边界
3. **轻量智能** — 总结和关键词提取不使用本地模型，预留 LLM 调用接口（可插拔）
4. **Project DNA 渐进更新** — 每次会话结束后增量更新，而非全量重建
5. **无状态 API** — 标准 REST 风格，方便 OpenHands/OpenClaw 通过 HTTP 直接调用
6. **API Only** — 不考虑本地和服务器跑模型，只考虑 API 接口

## 演进路线

- **Phase 1** — MBclaw-Lite：FastAPI + SQLite 长期记忆核心（已完成）
- **Phase 2** — 接入 OpenHands / OpenClaw / LangGraph
- **Phase 3** — Android APK 客户端（miclaw 融合）
- **Phase 4** — 多模型协作、自动执行 Agent
