# MVP v2 — 重定义边界

**版本**: v2
**日期**: 2026-06-21
**取代**: `docs/09-mbclaw-full-vision.md` 的"13 项目"方案

---

## 0. MVP 单一目标（必须能用一句话验证）

> **"用户连续 3 次会话后，第 4 次开口，AI 能正确复用前 3 次的关键事实、成功方案、失败教训。"**

如果做不到，其他都是装饰。

---

## 1. 范围圈定

### 必做（R0-R1）— 「记忆闭环」
| # | 能力 | 验证标准 |
|---|---|---|
| M1 | 会话持久化 | 每条消息落 SQLite + JSONL transcript |
| M2 | 会话总结 | session 结束触发 LLM 摘要，落库 |
| M3 | 关键词提取 | jieba + 简单 TF-IDF，落库 |
| M4 | 跨会话检索（L1） | 关键词 + SQLite FTS5，新会话开始时预注入 |
| M5 | Project DNA 增量更新 | goals / successful / failed 三字段 |
| M6 | MEMORY.md 双态读写 | 沿用 Lite 现有 memory_store 设计 |
| M7 | 写入审批（最简化） | 只保留"add/replace/remove"三种操作 + 一个全局阈值；删除多维度评分 |

### 延期（R2）— 「智能增强」
| # | 能力 | 触发条件 |
|---|---|---|
| L1 | 向量检索（ChromaDB） | M4 上线后真实查询命中率 < 70% 才启用 |
| L2 | 树状分类 | 数据量 > 100 sessions 才启用 |
| L3 | Curator 自动归档 | 数据量 > 500 entries 才启用 |
| L4 | Agent Runtime（工具调用） | 用户明确提出自动化需求时启用 |

### 延期（R3）— 「平台与生态」
| # | 能力 | 触发条件 |
|---|---|---|
| P1 | 11 平台消息网关 | 单平台跑通且有真实用户后扩展 |
| P2 | i18n 完整支持 | 海外用户出现后启用 |
| P3 | K8s 部署 | 真正有运维需求时启用 |
| P4 | Android APK / miclaw 融合 | Lite 服务端稳定 3 个月后启动 |

### **永久移出 Core**（仍可在 Design 探讨）
| # | 模块 | 理由 |
|---|---|---|
| X1 | Thought Collision（思维碰撞） | 创意产品，非记忆基础能力 |
| X2 | Utopia / 乌托邦计划 | 产品方向探索，不属于记忆系统 |
| X3 | Psychology Engine（心理画像） | 评分体系无科学依据，伪能力 |
| X4 | Dual-Key 协作循环 | 成本翻倍，无验证收益 |
| X5 | Sub-Agent Coordinator | 当前没有任何 agent 在跑 |

## 2. MVP 完成定义（DoD）

R1 发布需要同时满足：
1. ✅ 上述 M1-M7 全部有**端到端测试**（不只是单元测试）。
2. ✅ 有一个**演示脚本**：3 次会话后，第 4 次会话能复用关键事实。
3. ✅ 所有 Core 模块加起来 < **3000 行 Python**（当前 10379，需删 70%）。
4. ✅ 单一 `app/` 入口，无虚构子模块。
5. ✅ 文档与代码 1:1 同步，删除所有"已完成"但无证据的声明。

## 4. 不在 MVP 但保留的关键决策

- **API Only**（不在本地跑 LLM） — 战略不变。
- **SQLite 单库** — 短期不上 Postgres。
- **JSONL transcript** — 不改。
- **MEMORY.md 双态架构** — 不改（这是真亮点）。

## 6. CTO 决断

> 不要保留任何"未来可能有用"的代码进 Core。
> 想保留 → 进 Design（讨论）→ 进 Memory（归档）。
> Core 只允许"现在能跑通且有验证"的东西。
