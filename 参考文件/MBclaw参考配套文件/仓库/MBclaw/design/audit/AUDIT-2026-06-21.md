# MBclaw 项目审计报告

**日期**: 2026-06-21
**审计人**: Claude（CTO 角色）
**评估对象**: MBclaw-Lite 代码 + MBclaw 设计文档

---

## 1. 一句话结论

> **MBclaw-Lite 严重过度建设**：在 MVP 都未明确的阶段，已堆叠 39 个 service / 27 个 router / 24 个数据模型 / 10379 行代码，但核心"经验沉淀"闭环没有 端到端验证。
> **必须立刻冻结新增功能，进入收敛重构期。**

---

| 指标 | 数值 | 评估 |
|---|---|---|

## 3. 核心问题（按严重度排序）

### 🔴 P0：MVP 边界缺失
- `docs/01-vision.md` 列了 5 阶段路线图，但**没有一项是可以独立运行的最小闭环**。
- 已实现的 33 个"项目"互相耦合，**无法回答"删掉哪些还能跑"**。
- 后果：每个新需求都触发全栈改动；OpenHands 拿不到清晰任务。

### 🔴 P0：过度建模驱动复杂度
24 个模型中，**至少 9 个在 MVP 阶段毫无必要**：
| 模型 | 服务 | 真实价值 | 建议 |
|---|---|---|---|
| `Utopia*` (4个模型) | utopia_service / chat_extractor | "乌托邦计划"——产品方向，非记忆系统 | **移出 Core**，归档 Design |
| `ThoughtCollision` | collision_engine | 创意组合，非记忆基础能力 | **移出 Core** |
| `UserProfile / PositivePattern` | psychology_engine | 心理画像，伪科学评分 | **移出 Core** |
| `ExternalIntegration` (11 platforms) | integration_service | 在 MVP 阶段就接 11 个 IM 平台 | **延期至 R3** |
| `PendingApproval / ApprovalLog` | approval_gate | "写入审批门"——给虚构的 agent 设计 | **延期至 Agent 真正能写时** |

### 🟡 P1：分层搜索（L1/L2/L3）过早优化
- 同时实现 jieba 关键词 + TF-IDF + ChromaDB 向量。
- 当前数据量 0（无真实使用），三层混合策略**没有数据支撑**。
- 建议：MVP 只保留 L1（关键词）+ SQLite FTS5；向量层 R2 再加。

### 🟡 P1：Agent Runtime 设计先于使用场景
- `agent_runtime.py` 438 行，定义了 execution loop / tool executor / self-correction。
- 但**没有任何已验证的 Agent 用例**——没人调用过它产出真实价值。
- 风险：这是"先造引擎再找车"的典型反模式。

### 🟡 P1：测试集中在单文件
- 102 个测试全在 `tests/test_api.py`（2146 行）。
- 没有 service 单测、没有集成测试、没有失败场景覆盖。
- 改一个 service 不知道影响范围。

### 🟢 P2：i18n / 多平台网关 / K8s 部署
- 全部属于"过早工程化"，MVP 阶段都不需要。
- 不删除，但移出 Core 主干，标记 R3+。

## 4. 仍可保留的优秀决策（不要丢）

1. **FastAPI + SQLite + SQLAlchemy** — 选型正确，零负担。
2. **JSONL transcript + fcntl 写锁** — 朴素可靠。
3. **MEMORY.md 双态架构**（snapshot vs live） — 为 LLM 前缀缓存设计，有真实价值。
4. **会话即边界**的总结/关键词触发模型 — 概念清晰。
5. **API Only**（不在本地跑模型） — 战略正确。
6. **空闲调度器**触发后台整理 — 设计合理（实现要简化）。

---

## 5. 关键事实修正（文档 vs 代码）

| 文档声明 | 实际情况 | 差距 |
|---|---|---|
| "11,721 行代码 / 123 文件 / 137 tests" (`docs/zh/11`) | `app/` 10379 行 / 102 tests | 不一致 |
| "33/34 项目完成 ✅" | 大量"完成"项无端到端验证 | **"完成"≠"可用"** |
| "Stage A 完成 8/8，39 tests"（AGENTS.md） | 测试无法对应到 Stage 标签 | 缺乏映射 |
| "L1→L2→L3 已实现" | 实现存在，但无真实查询日志、无效果评估 | 形式化完成 |

**建议**：今后所有"完成"必须挂三个证据——①端到端测试；②实际调用日志；③用户/调用方反馈。
