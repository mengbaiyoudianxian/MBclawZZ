# 失败模式：文档声称完成度 ≠ 代码事实

## 表现

`docs/zh/11-implementation-status.md` 中：
- 声称 "123 Python 文件 / 11,721 行 / 137 tests"
- 实际审计：`app/` 10379 行 / 102 tests / 测试集中单文件
- 声称 "33/34 项目完成 ✅"
- 实际：多数"完成"仅有代码无端到端验证

## 根因

1. "完成"定义缺失——没有 DoD（Definition of Done）。
2. 文档由设计阶段写，未在实现后修订。
3. 没有 CI 校验"声称数 vs 实测数"。

## 修复（写入设计原则）

新 DoD（在 MVP-v2 §2 落地）：
- 端到端测试通过
- 演示脚本可执行
- 文档与代码一致性 CI 检查（R1 加）

## 教训

> "看起来在涨"的项目里，最隐蔽的债务是文档撒谎。
> 自我汇报必须永远附证据，否则把团队带进幻觉。
