# MBclaw 初始想法蓝图

> 关于借鉴 OpenClaw 的制作 MBclaw 方案（借鉴而不是改装）

---

## 项目一：详细日志系统
- 包含改了什么、用户与AI对话内容、AI思考了什么，全部详细备份
- 每个文件设定合适大小（方便后续整理分类）

## 项目二：空闲时自动分析整理
- 调用API自动分析整理备份的全部对话
- 根据对话类型主题做分类，变成分支结构
- 从粗略总结到详细内容，包含行不通的方案细节
- 给每个对话内容总结关键词，AI可根据关键词搜索唤醒详细记忆

## 项目三：突破时备份
- 在项目有突破时进行备份，以防后期修改导致的破坏

## 项目四：全自动模式
- 用户说"全自动"时，遇到需要确认的自己思考选最优解执行
- 执行完用户没反应，自动研究其他选择，尽可能做出几个成品让用户选

## 项目五：双Key协作
- Key1做产品，Key2来分析选择
- Key2评价Key1的产品给出改进方案、找Bug
- Key1继续改，循环1-6次

## 项目六：实时知识调用
- AI思考时根据用户对话内容，实时预调用项目2总结的知识
- 根据对话继续细节分析，调用更加精准的信息

## 项目七：用户最新消息优先
- 时时刻刻以最新用户信息为准
- 处理新消息时不停掉原来任务，放到后台进行

## 项目八：语言优化
- 报错命令翻译为用户选择语言并详解
- 默认配置界面详解翻译相应语言

## 项目九：启动检查
- 各种启动检查、安全检查、配置检查
- 冲突时优先保证MBclaw可启动，告诉用户报错项目

## 项目十：多子对话协同
- 子对话完成任务后自动反思，把结论发布到共享通道
- 其他子对话收到后直接复用，不重复查
- 遇到矛盾自动协商

## 项目十一：三层工具索引
- 轻量摘要 → 标签 → 完整描述
- 向量化语义搜索找工具
- Token预算控制：只注入相关工具

## 项目十二：模型-工具联合优化
- 多维模型能力评分（0-1分值）
- 成本感知调度（便宜模型优先）
- 模型-工具兼容性矩阵
- 工具选择和模型选择联合优化

## 项目十三：多编程工具融合
- OpenHands、MiMo Code等编程程序
- 图像识别、智能家居、语音转文字、文字转语音、路由器控制

## 项目十四：模拟思维碰撞
- 随机选取多个成功记忆和被肯定的技能
- 思考这些成果能组成什么，保留有价值的组合

## 项目十五：乌托邦计划
- 向用户申请读取分析聊天记录（QQ/微信/飞书/企业微信）
- 搜索关于智能体的相关信息（吐槽/夸奖/评价/Bug/想要的技能）
- 脱敏后归类上传服务器，服务器AI根据出现频率和刚需程度选择急需处理的
- 分发给用户端进行思考创作
- 智能体自评30% + 用户评价80%，综合>50%传回服务器收录

## 项目十六：自我进化
- 依靠项目十三，不断改进自身源代码
- 将自我进化数据匿名化上传服务器，用于母体成长

## 智能体之手 —— 视觉交互方案

### 设计原则
- 像人一样看：先看整体，再盯局部，不暴力分割
- 像人一样点：知道按钮大概在哪，也知道精确点哪
- 越用越准：每个设备的屏幕特性会被记住，每次偏差会被修正

### 四层架构
截图采集层 → 视觉理解层（粗筛+精定位） → 坐标映射层（归一化→物理像素+偏移校准） → 执行层（ADB/模拟点击）

### 屏幕标定
- 归一化坐标 + 设备级偏移校准
- 四点标定法（左上、右上、左下、右下）
- 自动微调机制：每次点击后对比前后截图，记录偏差自动修正

### 区块识别（双阶段）
- 粗筛：截图分割N×M网格，轻量模型判断目标在哪个格子
- 精定位：裁剪候选区域，高精度模型输出精确坐标
- 极端情况降级到全屏直接定位

### 快速模糊点击
- 方案A：轻量级YOLO检测模型，关键词库匹配（确认/取消/发送等），<500ms
- 方案B：视觉特征向量匹配，与缓存的典型界面特征库做相似度匹配
- 关键词库自动扩充

### 三维融合定位
- 优先级：模糊点击 > 精定位 > 粗筛
- 置信度融合加权平均，阈值0.8
- 冲突处理：坐标差距>10%时做二选一
- 执行后验证：对比前后截图

### 参数体系
- 极速模式：网格3×4，精定位0轮，模糊点击优先
- 均衡模式（默认）：网格4×6，精定位1轮，模糊点击辅助
- 高精度模式：网格6×8，精定位2轮，模糊点击关闭

### 异常处理与自愈
- 坐标在屏幕外 → 裁切到边界内
- 连续两次差距过大 → 暂停，评论家模型判断
- 界面无变化 → 小范围偏移重试
- 屏幕旋转 → 重新截图
- API报错 → 切换备用模型或历史坐标
- 同操作失败3次以上 → 自动调整执行策略

## MBclaw 最终构想

### 满血状态
- 已刷入MBclaw改装后的专属镜像
- 拥有Linux服务器 + Win设备专属客户端
- 主动化智能：主动收集用户习惯、喜好、思维特点
- 检测到可优化操作时主动介入（通知栏提示）
- 提供多种介入模式：屏幕辅助/后台操作/部分辅助/完全不需要

### 索引选择
- 中国IP：阿里云(text-embedding-v3)或智谱(embedding-3)
- 外国IP：embedding模型建索引
- 跳过则改为纯文fts-only（纯全文搜索）
